# AI Workbench Obsidian Vault

Este diretório pode ser aberto como *vault* no Obsidian.

- É uma camada de conhecimento auditável que complementa o repositório Git.
- **Nenhum segredo, token, .env, chave privada ou dado sensível de clientes** deve ser guardado aqui.
- Notas em `00_Inbox` são rascunhos (drafts) que ainda não foram revisados.
- Decisões aprovadas e regras operacionais críticas devem ser refletidas no Git (e no arquivo `HERMES.md`).
- Skills executáveis versionados ficam em `skills/hermes`.
- Runbooks versionados ficam em `docs`.
- O vault complementa, mas **não substitui**, o Git e o documento `HERMES.md`.

Cada pasta contém um `README.md` explicando rapidamente seu propósito.

## Operational rules

- This directory can be opened as vault in Obsidian.
- This vault must contain no secrets, tokens, `.env` files, private keys, or sensitive client data.
